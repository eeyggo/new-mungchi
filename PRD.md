# 뭉치(Mungchi) MVP 제품 요구사항 정의서 (PRD)

## 1. 제품 개요 (Product Overview)

*   **목적**: 대학 상권(고려대 등) 내 행정실, 회사, 병원, 학생회 등 **단체 식사/간식 주문**이 필요한 모든 수요층에게 적합한 식당 정보를 통합 제공하는 서비스.
*   **배경**: 기존에는 학생들이나 행정실 직원들이 단체 식당을 찾기 위해 검색과 전화 확인을 반복해야 했음. '뭉치'는 이를 해결하기 위해 검증된 식당 리스트와 직관적인 컨택 포인트를 제공하여 탐색 비용을 최소화함.

## 2. 비즈니스 목표 및 핵심 지표 (Business Goals & KPI)

*   **비즈니스 목표**: 고려대 주변 단체 주문 수요(학생+직장인)를 확인하고, 실제 식당 컨택(전화/지도확인)으로 이어지는 연결 고리를 검증.
*   **핵심 지표 (KPI)**:
    1.  메인 페이지 -> 식당 상세 페이지 전환율
    2.  '전화하기' 버튼 클릭 횟수
    3.  '지도 보기' 버튼 클릭 횟수

## 3. 핵심 고객 및 타겟 (Target Audience)

*   **핵심 타겟**: 고려대(안암) 인근에서 5인 이상 단체 식사를 기획하는 모든 사람.
    *   **대학생**: 동아리, 학회, 학생회 회장단 (가성비, 분위기 중시).
    *   **직장인/교직원**: 학교 행정실, 인근 회사, 병원 관계자 (법인카드 결제 가능 여부, 룸 유무, 주차 등 편의성 중시, 깔끔한 정보 선호).
*   **지역 거점**: 고려대학교(안암) 상권.

## 4. 디자인 컨셉 (Design Concept)

*   **키워드**: 깔끔함(Clean), 트렌디(Trendy), 신뢰감(Professional).
*   **레퍼런스**: `design/Mobile Web MVP UI Design` 폴더 내 UI 스타일을 기반으로 현대적이고 직관적인 모바일 웹 인터페이스 구축.

## 5. 기능적 요구 사항 (Functional Requirements)

### [Must Have] 필수 구현 기능

#### A. 메인 페이지 (식당 리스트)
*   식당 리스트를 카드 형태로 깔끔하게 나열.
*   **썸네일 + 식당 이름 + 대표 카테고리 + 태그(#)** 노출.
*   빠른 탐색을 위한 직관적 UI.

#### B. 식당 상세 페이지
*   선택한 식당의 상세 정보 제공 및 액션 유도.
*   **핵심 정보**:
    *   **이미지**: 식당/음식 대표 사진.
    *   **이름**: 식당 상호명.
    *   **주소**: 식당 위치(텍스트).
    *   **메뉴 카테고리**: (예: 한식/중식/도시락).
    *   **영업시간**: 운영 시간 정보.
    *   **단체 주문 경험(간식행사)**: 해당 식당이 단체 주문/간식 행사를 진행해본 경험이 있는지 여부.
    *   **특이사항/추천사**: (예: "지난번 행정실 행사 때 반응 좋았음", "사장님 친절함").
    *   **태그(#)**: (예: #가성비, #룸완비, #주차가능).
    *   **전화번호**: 클릭 시 바로 연결.
*   **Action Buttons**:
    *   **네이버 지도 보기**: 외부 링크 이동.
    *   **전화하기**: `tel:` 스키마 연결.
